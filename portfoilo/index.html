시트 신규 기능 개발 및 개선

엑셀 업로드 다이얼로그

  • 선택한 파일을 시트에 업로드 전, 다이얼로그를 통해 사용자가 데이터를 확인하고 핸들링 할 수 있는 기능
  • 설계:
    1) 시트 업로드 전, 다이얼로그를 통해 프리뷰 시트(실제 시트와 동일한)를 하나 만듬.
    2) 프리뷰 시트에서 핸들링되는 동작들이 반영 되도록.. (추가, 숨김, 삭제, 데이터 수정.. 등)
    3) 시트와 엑셀을 맵핑 시키기 위해, 시트의 컬럼 갯수가 엑셀 보다 적지 않도록 여분의 더미 컬럼을 생성시킴.
    4) 다른 txt, csv 파일도 업로드가 가능함으로, 이들의 데이터는 각 구분자를 사용자가 직접 선택할 수 있도록.
  • 결과:
    1) 사용자에게 GUI 기능을 지원하며 사용자의 편의성을 극대화.
    2) 중간에서 핸들링을 통해 시트와 엑셀의 데이터가 맞지 않아도 맵핑 시킬 수 있음.
    3) txt, csv 업로드를 각 구분자를 통해 변환 시킬수 있는 기능을 만듬으로써, 사용자가 구분자를 신경 쓰지 않아도 됨.

클라이언트 엑셀 다운로드, 업로드

  • 엑셀 사용시 서버모듈을 사용하여 다운로드나 업로드를 처리하는데, 서버를 쓰지 못하는 환경에서도 엑셀을 사용할 수 있는 기능
  • 상황: 1) 엑셀 다운로드 - 기존의 개발자가 JsZip을 이용하여 다운시에 시트의 틀이 어느정도 나오도록 해서 넘겨 받음. 2) 엑셀 업로드 - 처음부터 개발하여 여러 라이브러리를 테스트 해보고 SheetJS 라이브러리를 사용하여 개발함.
  • 다운로드 설계:
    1) 기존 제공하는 엑셀 기능들 제공(파일명, 다운 받을 행, 열, 제목기능, 머지).
    2) 엑셀 다운시에 현재 보이는 포멧 형태를 가져오도록.
    3) 다운시 현재 스타일을 가져오도록.
  • 다운로드 결과:
    1) 기존에는 서버에서 제공하는 기능들을 제공함.
    2) 데이터가 적거나 스타일이나 포멧이 많은 경우, 서버보다 높은 속도를 자랑함.
    3) IE 10 이상에서만, 사용가능 하고 xls 같은 경우 완벽하게 지원이 어려움.
  • 업로드 설계:
    1) 기존 제공하는 엑셀 기능들 제공(파일명, 다운 받을 행, 열, 제목기능, 머지).
    2) 엑셀 업로드시에 엑셀에서 시작하는 컬럼과 행을 선택 할 수 있도록.
    3) 업로드시 인코딩이 필요한 경우, 원하는 인코딩을 할 수 있도록.
    4) 업로드 후에 처리할 수 있는 동작 필요.
  • 업로드 결과:
    1) 기존 제공하는 기능들을 제공함.
    2) 업로드 후처리 동작을 할 수 있도록 이벤트를 제공.
    3) 엑셀의 시트가 있는 위치를 찾아 업로드 할 수 있도록 제공.
    4) IE 이상에서만, 사용가능함.
  • 피드백: 사용자가 엑셀 다운 및 업로드시에 서버모듈, 클라이언트 모듈을 자동으로 선택할 수 있는 기능.

Enter를 이용하여, 포커스가 되는 기능 확장성 개선

  • Enter를 눌러 포커스가 어떤 식(이동 방향, 포커스가 가능한지..)으로 될지에 대한 기능.
  • 문제: 사용자가 엔터를 눌러 포커스 하는 방식이 각기 다르고, 현재 기능에 하나씩 추가하자니 양이 너무 많음.
    각 기능별로 쓰는게 아니라, 원하는 방식을 붙여서 커스터마이징 하고 싶음.
  • 설계:
    1) 기존의 사용자가 쓸 수 있도록 호환성 유지
    2) 비트 연산으로 제공하여 사용자가 여러 기능을 비트 연산으로 중복해서 사용할 수 있도록.
  • 리뷰:
    1) 기존의 사용자가 쓸 수 있도록 기존의 형태와 새로운 형태 두 가지를 지원함.
    2) 비트 연산을 제공함으로써 해당 모드가 끝없이 많아질 수 있는 현상을 기존 모드 + 옵션(비트 연산) 으로 제공 함으로써
    유저의 커스터마이징의 자유로를 올리고, 불필요한 노가다 성 코드가 사라짐.

조회함수에 대한 콜백 추가

    2) 렌더링 비동기 로직까지 고려하여 최종적인 비동기가 완료되는 정확한 시점 캐치.
  • 리뷰:
    1) 정확한 callback 시점으로 동작들이 정상적으로 작동.
