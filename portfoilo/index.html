사용자 편집시 소계 계산 성능 개선

  • 문제:
    소계가 있는 경우에, 사용자가 편집시에 소계가 계산에 성능이 꽤 많이 잡아먹히는 문제.
    편집시 렌더링 완료까지 약 0.8초 소비
  • 원인:
    1) 편집 이후, 변화된 값에 대해 소계에 대해 전체 계산
    2) 불필요한 렌더링 로직.
  • 해결방법:
    1) 편집 이후, 변화된 값에 대해 전체 계산을 하지 않고, 소계 해당 그룹만 계산할 수 있게끔 개선.
    2) 계산 이후, 셀을 렌더링하는 구간을 여러번 탔는데 계산이 전부 완료된 이후 마지막에만 조정할 수 있도록 수정.
  • 평가:
    편집시 렌더링 완료까지 0.8 초에서 0.3 초 수준으로 줄었음.

여러 행들을 지우는 API 성능 개선

  • 문제:
    500개의 행을 지우는 경우, 5초이상 소비되고 갯수가 많아질수록 성능은 심각할정도로 떨어짐.
  • 원인:
    1) 행을 삭제할 때 마다 불필요한 렌더링을 하다보니, 많은 강제 리플로우가 발생하면서 리페인팅이 계속해서 발생함.
  • 해결방법:
    1) 행을 삭제할 때 마다 렌더링을 하지 않고 계산만 해주고 마지막에 행을 지운 뒤, 한 번만 렌더링을 할 수 있도록 개선함.
    2) 중복된 계산 로직을 제거.
  • 평가:
    행을 삭제하고 최종 렌더링 까지 5초 => 0.8초 이하로 속도를 개선함.

시트가 화면에 보이지 않을시에 대한 고려.

  • 문제:
    화면에 보이는 만큼만 그리는 모드를 지원하면서, 시트가 display:none 인 경우에 시트가 화면에 보이지 않아 그리지 않음으로 발생하는 문제
  • 원인:
    1) 화면에 보이지 않을시에 페인트 작업을 하지 않는데, 해당 요소에 직접 접근하는 API가 있음.
    2) display를 동적으로 움직였을시에, 리프레쉬 작업이 제대로 일어나지 않아 오류가 발생함.
  • 해결방법:
    1) 화면에 보이지 않을시에 요소에 접근하는 방식이 아닌, 객체에 접근하는 방식을 추가
    2) setInterVal을 사용하여, 현재 상태가 화면에 보이는지 확인 후, 리프레쉬 작업을 할 수 있도록 수정.
  • 평가:
    화면에 보이지 않을시에 대한 로직 구조화.
    display를 토글할 시에 대해 대응.

필터링 이후, 행 추가시 행이 사라지는 현상

  • 문제:
    필터링 이후에, 행을 추가하고 바디영역 부분이 새로 렌더링이 발생할시에 추가된 행들이 없어지는 문제
  • 원인:
    1) 화면에 그려지는 행에 대해 관리하는 객체와 필터링 이후 필터가 된 행을 관리하는 객체가 서로 맵핑이 제대로 되지 않아 생기는 문제.
  • 해결방법:
    1) 각 객체의 정확한 역할을 분담하여 렌더링시에는 화면에 그려지는 행 객체를 보도록 하고, 필터링 관련 계산은 필터 관련 객체를 보도록 로직 재수정.
  • 평가:
    필터링 이후, 행 추가 동작말고 다른 동작시에도 렌더링 문제가 발생하는 문제가 사라짐.
